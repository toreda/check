Interface BlockBaseInit

Parameters used by rule blocks to intialize it their base block data.

Remarks

Normally two types with overlap like BlockInit and BlockBaseInit would be combined into a single type, or one would extend the other and add properties. Here they should remain separate types due to complications of using the InputT needed by BlockInit<InputT> in the base block extended by all blocks. BlockBaseInit doesn't need InputT.

interface BlockBaseInit {
    blockType: BlockType;
    modType?: BlockModType;
    name: string;
